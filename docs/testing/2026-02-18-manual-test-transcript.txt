  motion again. Each reset fires the automation.

  ---
  Root Cause

  PIR sensor re-trigger cycles. Your binary_sensor.downstairs_bathroom_motion is
   toggling off→on repeatedly during a single occupancy period. This is normal
  for PIR sensors — they have a cooldown window after each detection, during
  which motion briefly reads off before going on again when the next IR change
  is detected.

  The automation has no condition checking if the light is already on, so 
  light.turn_on fires on every motion pulse — even if the light is already lit.
  This is harmless (calling turn_on on an already-on light is a no-op), but it
  means the automation log shows multiple runs per bathroom visit.

  Note: HA only stores 5 traces per automation, so the morning runs you noticed
  were pushed out by the afternoon activity. The same re-trigger pattern
  explains both.

  ---
  Options if You Want to Reduce Noise

  1. Add a condition to the motion-on automation: only act if
  light.downstairs_bathroom is currently off. This cuts the run count without
  changing any visible behavior. Want me to generate that change via the
  ha-automations skill?
  2. Adjust the sensor's retrigger interval on the device itself (if it's a
  Zigbee/Z-Wave sensor, many have a configurable occupancy timeout in their
  settings). Longer cooldown = fewer pulses per visit.
  3. Accept current behavior — the light cycling you saw in the morning was
  likely the light going off after 15 min no-motion and then coming back when
  someone re-entered, not a bug.
